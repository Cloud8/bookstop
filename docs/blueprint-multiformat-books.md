# Blueprint — Multi-Format Books (Phase 13)

## Context

Books currently support a single EPUB file stored directly on the `books` table (`epub_path`). This phase introduces a proper multi-format system: EPUB + FB2 for clients, DOCX as an upload source that gets auto-converted. Conversions run as background jobs using Pandoc (DOCX→EPUB) or Calibre (all other pairs).

---

## Business Rules

### File Management
- **Rule 1**: A book may have multiple `book_files` records — one per format+role combination.
- **Rule 2**: `role=source` — the file uploaded by the admin (DOCX or EPUB). `role=derived` — files generated by conversion.
- **Rule 3**: DOCX is never accessible to clients. Only EPUB and FB2 with `status=ready` are downloadable.
- **Rule 4**: When a source file is uploaded, conversion jobs are dispatched automatically for all applicable target formats.
- **Rule 5**: A conversion job sets `status=processing` on start, `status=ready` on success, `status=failed` (with `error_message`) on failure.
- **Rule 6**: Admin can manually retry a failed conversion via a "Retry" button in the admin UI.

### Publish Guard
- **Rule 7**: A book can only be published (status=published) if it has at least one client-accessible `BookFile` (`format` IN (epub, fb2), `status=ready`). This is enforced in `BookObserver` or the admin controller before saving `status=published`.

### Downloads
- **Rule 8**: The download endpoint (`/books/{book}/download`) accepts a `format` query parameter (epub or fb2).
- **Rule 9**: Only `ready` files in client-accessible formats are shown in the download dropdown.
- **Rule 10**: DOCX is never exposed via download routes — triple gate: enum `isClientAccessible()`, controller validation, service layer.

### Conversion Matrix
- **Rule 11**: DOCX → EPUB: use Pandoc (better quality for prose).
- **Rule 12**: DOCX → FB2: use Calibre.
- **Rule 13**: EPUB → FB2: use Calibre.
- **Rule 14**: FB2 → EPUB is NOT supported (out of scope).
- **Rule 15**: The conversion matrix is defined in `config/bookshop.php`, not hardcoded in jobs or services.

### epub_path Migration
- **Rule 16**: The `epub_path` column on `books` is dropped. Existing data (if any) is migrated to `book_files` records before the column is dropped. On a fresh install (no existing books), the migration simply drops the column.

## Database Schema

### `book_files` table

```
id               bigint unsigned PK
book_id          FK → books.id (cascade delete)
format           enum: docx, epub, fb2
role             enum: source, derived

status           enum: pending, processing, ready, failed
```

## Models

### `App\Models\BookFile`

```php
class BookFile extends Model
{
    // fillable: book_id, format, role, path, status, error_message, converted_at
    // casts: format → BookFileFormat, status → BookFileStatus, converted_at → datetime

    public function book(): BelongsTo<Book, $this>;
    public function isReady(): bool;
    public function isSource(): bool;
    public function isClientAccessible(): bool; // delegates to format->isClientAccessible()
}
```

### `App\Models\Book` — additions

```php
/** @return HasMany<BookFile, $this> */
public function files(): HasMany;

/**
 * Returns true if the book has at least one client-accessible BookFile
 * (EPUB or FB2) with status=ready. Used as the publish guard.
 */
public function hasClientReadyFile(): bool;
```

Note: Do NOT add `readyFiles()`, `clientFiles()`, or other filtered HasMany pseudo-relations. Filtered relations behave unexpectedly under `load()` / `with()` with constraints. Use inline query builders instead (as in `hasClientReadyFile()`).

## Jobs

### `App\Features\Books\Jobs\ConvertBookFormat`

Queued job. One job per `BookFile` (derived).

```php
class ConvertBookFormat implements ShouldQueue
{
    public function __construct(
        public readonly BookFile $bookFile,
    ) {}

    public function handle(ConverterRegistry $registry, Storage $storage): void;
}
```

Flow:
1. Set `bookFile->status = processing`, save.
2. Download source file from S3 to temp path.
3. `$registry->for($sourceFormat, $targetFormat)->convert(...)`.
4. On success: upload output to S3, set `path`, `status=ready`, `converted_at=now()`.
5. On failure: set `status=failed`, `error_message=stderr`, save.
6. Clean up temp files in `finally`.

## Observers

### `App\Features\Books\Observers\BookFileObserver`

On `created` event for a source file:
- Determine target formats from `config('bookshop.conversion_matrix')[$sourceFormat]`.
- Create `BookFile` records with `status=pending, role=derived` for each target.
- Dispatch `ConvertBookFormat` jobs.

## Admin UI

On the book edit page — "Files" block:

```
Source:  book.docx   [ready]   [Replace]
EPUB:    —           [processing...]
FB2:     —           [failed: "ebook-convert exit 1: ..."]  [Retry]
```

- Upload new source → replaces existing source, triggers new conversions.
- Retry button → re-dispatches `ConvertBookFormat` for the failed `BookFile`.
